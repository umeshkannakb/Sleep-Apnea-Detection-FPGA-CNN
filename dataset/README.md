## Channels Used

Only two EEG channels are used from the full PSG recordings:

| Channel | Location | Significance |
|---------|----------|-------------|
| C3-A2 | Left central cortex | High sensitivity to cortical arousals during apnea |
| C4-A1 | Right central cortex | Bilateral cortical activity monitoring |

Both channels are sampled at **128 Hz** and combined into a [N x 2] matrix.

---

## Annotation Files

Each subject has a corresponding `timing_apnea.mat` file with:
- Start and end times (in seconds) of all apnea events
- Used to generate binary ground-truth labels:
  - `0` = Normal breathing (non-apneic)
  - `1` = Apnea event

Labels are stored per-second: `ucddb003_label.mat`

---

## Preprocessing Output Files

For each subject (e.g., `ucddb003`), the following MATLAB files are generated:

| File | Contents |
|------|---------|
| `ucddb003_C3A2.mat` | Raw C3-A2 EEG channel |
| `ucddb003_C4A1.mat` | Raw C4-A1 EEG channel |
| `ucddb003_label.mat` | Per-second binary labels (0/1) |
| `ucddb003_eeg_train.mat` | Training EEG segments [N_train x 2816] |
| `ucddb003_train_labels.mat` | Training labels [N_train x 1] |
| `ucddb003_eeg_valid.mat` | Validation EEG segments |
| `ucddb003_valid_labels.mat` | Validation labels |
| `ucddb003_eeg_test.mat` | Test EEG segments |
| `ucddb003_test_labels.mat` | Test labels |

## Segmentation Details

| Parameter | Value |
|-----------|-------|
| Sampling rate | 128 Hz |
| Window size | 11 seconds |
| Window overlap | 10 seconds |
| Stride | 1 second (128 samples) |
| Samples per segment | 2816 (11s x 128 Hz x 2 channels) |
| Labeling rule | Label = class of 2nd second in window |
| Minimum apnea duration | 10 seconds (clinical definition) |

## Class Distribution (Typical)

Sleep apnea datasets are heavily imbalanced. In a typical overnight recording:
- ~85-90% of epochs are Normal
- ~10-15% are Apnea

Two strategies are used to address this:

### Random Oversampling
Minority (apnea) samples are randomly duplicated with replacement until class sizes are equal. Implemented via MATLAB `datasample()`.

### SMOTE (Synthetic Minority Oversampling Technique)
Synthetic apnea samples are generated by interpolating between k-nearest neighbors in feature space. Produces more diverse minority samples and reduces overfitting risk.
